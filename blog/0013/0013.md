We experiment with formulating the paradoxical arguments for the undecidability of the halting problem and, more generally, Rice‚Äôs theorem in a modally typed setting using (a variation of) Kavvos‚Äôs intensional recursion, whose type corresponds to the G√∂del‚ÄìL√∂b axiom.

## Rice‚Äôs theorem, modally typed

My colleagues [Liang-Ting Chen](https://l-tchen.github.io) and [Tsung-Ju Chiang](https://dblp.org/pid/273/7936.html) and I have been looking into modal type theory in search of a solid theoretic foundation for typed meta-programming.
In particular, we have concentrated on (a variation of) [Alex Kavvos](https://www.lambdabetaeta.eu)‚Äôs *intensional recursion* (proposed [here](https://arxiv.org/abs/1703.01288)), a type-theoretic formulation of the G√∂del-L√∂b (GL) axiom.
Computationally, GL corresponds to Kleene‚Äôs second recursion theorem, which allows us to write programs that have access to their own code.
This self-referencing ability leads to paradoxical constructions that lie at the heart of the classic results in computability theory, for example the undecidability of the halting problem and, more generally, Rice‚Äôs theorem.
Below is our attempt to formulate the arguments somewhat informally with modal types, followed by some discussions of future directions.

We start with intensional recursion, which can be understood by comparing it with ‚Äòextensional‚Äô recursion, that is, the familiar fixed-point combinator that allows us to define recursive functions

```
Y : (Œ± ‚Üí Œ±) ‚Üí Œ±
```

which satisfies the equation

```
Y f = f (Y f)
```

To define something of type `Œ±` recursively with `Y`, we should supply a function of type `Œ± ‚Üí Œ±`, in which we define something of type `Œ±` in terms of the argument also of type `Œ±`, which is a reference to the thing being defined.
The type for intensional recursion has the same structure except that we throw in some modalities:

```
GL : ‚ñ°(‚ñ°Œ± ‚Üí Œ±) ‚Üí ‚ñ°Œ±
```

Think of `‚ñ°Œ±` as the type of ‚Äòcode representing something of type `Œ±`‚Äô.
Of the three `‚ñ°`‚Äôs above, the second one is the most essential because it signifies what kind of self-reference is permitted here: to define (the code of) something of type `Œ±` using `GL`, we supply (the code of) a function of type `‚ñ°Œ± ‚Üí Œ±`, in which we define something of type `Œ±` in terms of the argument ‚Äî this time of type `‚ñ°Œ±` ‚Äî which is a reference to *the code of* the thing being defined.
We can also write down a fixed-point equation for `GL`:

```
‚ü¶ GL c ‚üß = ‚ü¶ c ‚üß (GL c)
```

for all `c : ‚ñ°(‚ñ°Œ± ‚Üí Œ±)`, where `‚ü¶_‚üß : ‚ñ°Œ± ‚Üí Œ±` is the evaluator/interpreter function.
This equation gives code a coinductive feel: `GL c : ‚ñ°Œ±` by itself does not reduce, but if we ‚Äòforce‚Äô it by applying `‚ü¶_‚üß`, a copy of its body `c` will be forced/evaluated/expanded, and the result will refer to `GL c` itself, which will need to be forced again to spawn another copy of `c`, and so on.


An important fact has to be mentioned at this point: `GL` and `‚ü¶_‚üß` together give rise to `Y`, and thus non-termination.
Here is the construction, which also serves as a first example of programming with `GL`:

```
Y : (Œ± ‚Üí Œ±) ‚Üí Œ±
Y = ‚ü¶ GL ‚åúŒªself f. f (‚ü¶ self ‚üß f)‚åù ‚üß
```

Here `‚åú_‚åù` quotes (at meta-level) a term of type `Œ±` and turns it into a piece of code that has type `‚ñ°Œ±`; it interacts with evaluation via the equation `‚ü¶ ‚åút‚åù ‚üß = t`.
We‚Äôre defining the `Y` combinator using `GL`, so we can suppose that we‚Äôre given the code `self : ‚ñ°((Œ± ‚Üí Œ±) ‚Üí Œ±)` of the combinator itself, and then define the combinator in terms of `self`.
Onto the actual meat of the definition: the combinator should take a function `f : Œ± ‚Üí Œ±` and compute the infinitely nested applications of `f`, so in the definition we receive a second argument `f` and make the outermost invocation of `f`, whose argument is computed by evaluating `self` ‚Äî the code of the combinator ‚Äî on `f`, producing the rest of the infinitely nested applications.
It is straightforward to verify that this definition does give rise to the intended behaviour of extensional recursion:

```
  Y f
=  {- above definition of Y -}
  ‚ü¶ GL ‚åúŒªself f. f (‚ü¶ self ‚üß f)‚åù ‚üß f
=  {- fixed-point equation of GL -}
  ‚ü¶ ‚åúŒªself f. f (‚ü¶ self ‚üß f)‚åù ‚üß (GL ‚åúŒªself f. f (‚ü¶ self ‚üß f)‚åù) f
=  {- evaluating a quotation; function application (twice) -}
  f (‚ü¶ GL ‚åúŒªself f. f (‚ü¶ self ‚üß f)‚åù ‚üß f)
=  {- above definition of Y -}
  f (Y f)
```

Now we‚Äôre ready to establish the undecidability of the halting problem with `GL`.
Suppose that we‚Äôre given a total decision procedure `h : ‚ñ°A ‚Üí Bool` such that `h c = true` iff `‚ü¶ c ‚üß ‚Üì` (converges) for all `c : ‚ñ°A`.
We can then construct a ‚Äòdisobedient‚Äô program `GL oppositeH : ‚ñ°A` that does the opposite of what `h` says it does.
Assuming the existence of a convergent value `stop : A`, it is tempting to define the body `oppositeH : ‚ñ°(‚ñ°A ‚Üí A)` as the quote of the function 

```
Œªself. if h self then Y id else stop
  : ‚ñ°A ‚Üí A
```

which says that if `h` says it‚Äôs convergent then it will become `Y id` and diverge, or otherwise it will become `stop` and converge.
However, the above expression is an open one (whose free variables are the assumptions `h` and `stop`), and quotation acts on closed expressions only (unless we switch to a contextual modal type theory, which we avoid for now).
So to be technically correct, we should instead construct a closed expression

```
Œªh stop self. if h self then Y id else stop
  : (‚ñ°A ‚Üí Bool) ‚Üí A ‚Üí ‚ñ°A ‚Üí A
```

and define `oppositeH` by ‚Äòapplying‚Äô the quote of the expression above to our two assumptions using the applicative combinator

```
_¬∑_ : ‚ñ°(Œ± ‚Üí Œ≤) ‚Üí ‚ñ°Œ± ‚Üí ‚ñ°Œ≤
```

which interacts with evaluation via the equation `‚ü¶ cf ¬∑ cx ‚üß = ‚ü¶ cf ‚üß ‚ü¶ cx ‚üß`.
This means that the assumptions should also be `‚ñ°`-typed ‚Äî our real assumptions are thus

```
ch    : ‚ñ°(‚ñ°A ‚Üí Bool)
cstop : ‚ñ°A
```

such that `‚ü¶ ch ‚üß` is total, `‚ü¶ ch ‚üß c = true` iff `‚ü¶ c ‚üß ‚Üì`, and `‚ü¶ cstop ‚üß ‚Üì`.
Assuming that `_¬∑_` is left-associative, the definition of `oppositeH` is then

```
oppositeH : ‚ñ°(‚ñ°A ‚Üí A)
oppositeH = ‚åúŒªh stop self. if h self then Y id else stop‚åù ¬∑ ch ¬∑ cstop
```

Now we can check that `GL oppositeH` is indeed disobedient and leads to a paradox and eventually a contradiction:

```
  ‚ü¶ ch ‚üß (GL oppositeH) = true
‚áî  {- ‚ü¶ ch ‚üß decides the halting problem -}
  ‚ü¶ GL oppositeH ‚üß  ‚Üì
‚áî  {- fixed-point equation of GL -}
  ‚ü¶ oppositeH ‚üß (GL oppositeH)  ‚Üì
‚áî  {- definition of oppositeH; evaluating code application -}
  if ‚ü¶ ch ‚üß (GL oppositeH) then Y id else ‚ü¶ cstop ‚üß  ‚Üì
‚áî  {- (only) the else branch converges -}
  ‚ü¶ ch ‚üß (GL oppositeH) = false
```

Note that if `‚ü¶ ch ‚üß` was not total, in particular if `‚ü¶ ch ‚üß` was not defined on `GL oppositeH`, then this bi-implication wouldn‚Äôt be a contradiction, so the totality assumption about `‚ü¶ ch ‚üß` is important ‚Äî given that `‚ü¶ ch ‚üß (GL oppositeH)` is defined, it is either `true` or `false`, but the argument above says that it has to be both, which is impossible.
Also important is the assumption that `cstop` exists.
Due to the presence of `Y`, we know that the type `A` necessarily has divergent values, but we can‚Äôt say the same thing for convergent values, and must assume their existence so that the construction can go through.
This assumption can be phrased alternatively as ‚Äò`A` has to contain both divergent and convergent values‚Äô, which is closer to the form of the ‚Äònon-triviality‚Äô assumption of Rice‚Äôs theorem, which we will see next.

In the above construction, whether we are dealing with convergence or some other property doesn‚Äôt seem crucial ‚Äî the essence of the construction is to do the opposite of what a given decision procedure `‚ü¶ cd ‚üß : ‚ñ°A ‚Üí Bool` (the evaluation of some `cd : ‚ñ°(‚ñ°A ‚Üí Bool)`) says, so we can carry out the construction as long as the property decided by `‚ü¶ cd ‚üß` is *non-trivial*, that is, there exist `ca : ‚ñ°A` and `cb : ‚ñ°A` such that `‚ü¶ cd ‚üß ca = true` and `‚ü¶ cd ‚üß cb = false`.
The generalised construction is then

```
oppositeR : ‚ñ°(‚ñ°A ‚Üí A)
oppositeR = ‚åúŒªd a b self. if d self then b else a‚åù ¬∑ cd ¬∑ ca ¬∑ cb
```

For the paradoxical argument to go through, however, we need one more assumption about `‚ü¶ cd ‚üß`: it should be *extensional*, that is, `‚ü¶ cx ‚üß = ‚ü¶ cy ‚üß` implies `‚ü¶ cd ‚üß cx = ‚ü¶ cd ‚üß cy` for all `cx : ‚ñ°A` and `cy : ‚ñ°A` ‚Äî the decision procedure should say the same thing for extensionally equal programs, i.e., programs with the same behaviour, or contrapositively, if two programs are distinguished by the decision procedure, they must have different behaviour.
Now we can largely replay the argument:

```
  ‚ü¶ cd ‚üß (GL oppositeR) = true
‚áî  {- (‚áí) see below; (‚áê) ‚ü¶ cd ‚üß extensional and ‚ü¶ cd ‚üß ca = true -}
  ‚ü¶ GL oppositeR ‚üß = ‚ü¶ ca ‚üß
‚áî  {- fixed-point equation of GL -}
  ‚ü¶ oppositeR ‚üß (GL oppositeR) = ‚ü¶ ca ‚üß
‚áî  {- definition of oppositeR; evaluating code application -}
  if ‚ü¶ cd ‚üß (GL oppositeR) then ‚ü¶ cb ‚üß else ‚ü¶ ca ‚üß = ‚ü¶ ca ‚üß
‚áî  {- (only) the else branch equals ‚ü¶ ca ‚üß -}
  ‚ü¶ cd ‚üß (GL oppositeR) = false
```

In the forward direction of the first step, by the fixed-point equation of `GL` and the totality of `‚ü¶ cd ‚üß`, we see that `‚ü¶ GL oppositeR ‚üß` is either `‚ü¶ ca ‚üß` or `‚ü¶ cb ‚üß`, but the second case is impossible since `‚ü¶ GL oppositeR ‚üß = ‚ü¶ cb ‚üß` would imply `‚ü¶ cd ‚üß (GL oppositeR) = ‚ü¶ cd ‚üß cb = false` by the extensionality of `‚ü¶ cd ‚üß`, contradicting the antecedent.
The extensionality of `‚ü¶ cd ‚üß` also plays a role in the forward direction of the last step, where we need `‚ü¶ ca ‚üß ‚â† ‚ü¶ cb ‚üß`, which holds because `ca` and `cb` are distinguished by `‚ü¶ cd ‚üß` and thus cannot be extensionally equal.
What we‚Äôve proved is exactly Rice‚Äôs theorem: non-trivial and extensional decision procedures cannot exist ‚Äî it is impossible to write a decision procedure that distinguishes programs in terms of their behaviour.

The proofs and even the statements of the undecidability of the halting problem and Rice‚Äôs theorem presented here are different from those in existing literature, though.
Traditional treatments are untyped and thus are able to construct an untyped self-application that leads to the paradox; the relationship between the untyped arguments and the typed ones presented here is still mysterious to me.
Another mysterious relationship to be investigated is the one between the presented arguments, which use intensional recursion, and a categorical version of Rice‚Äôs theorem presented by Kavvos ([here](https://doi.org/10.1007/978-3-662-54458-7_32)), which uses extensional recursion.

And Rice‚Äôs theorem is still not intuitive enough to me, probably because it‚Äôs a negative theorem ‚Äî what it constructs is a contradiction.
In constructive mathematics we often prefer positive notions to negative ones ‚Äî for example, instead of defining inequality as the negation of equality, which only says that it‚Äôs impossible for two things to be equal, it is preferable to use an *apartness* relation that positively provide evidence that separates two things.
What would a positive version of Rice‚Äôs theorem look like?

The theoretical foundation of the development above is essentially [Kavvos‚Äôs *intensional PCF*](https://arxiv.org/abs/1703.01288), which (like PCF) is not strongly normalising (since we can derive `Y`).
If what we want is a strongly normalising calculus with `GL`, then we can‚Äôt allow the calculus to be able to express evaluation, which, however, seems to be essential to the whole development.
We‚Äôre now working on an alternative calculus (which should be strongly normalising) with two code modalities `‚ñ°` and `‚ä†` (based on [this paper](https://doi.org/10.1016/j.apal.2009.07.020)), where `‚ñ°` marks code for self-reference but not for evaluation, and `‚ä†` marks code that can be evaluated.
By ‚Äòupgrading‚Äô `GL` to the type `‚ä†(‚ñ°Œ± ‚Üí Œ±) ‚Üí ‚ä†Œ±` and including a combinator of type `‚ä†Œ± ‚Üí ‚ñ°Œ±` in this calculus, we can type the construction for Rice‚Äôs theorem, which doesn‚Äôt evaluate the `‚ñ°`-typed argument `self`, but not `Y`, which evaluates `self`.
In this calculus, the halting problem ceases to be meaningful because all programs necessarily terminate.
For Rice‚Äôs theorem to be meaningful, we need to make sure that the type `‚ñ°A ‚Üí Bool` is inhabited by non-constant functions so that the statement of Rice‚Äôs theorem properly quantifies over non-trivial decision procedures.
That is, we should add to the calculus interesting operations on `‚ñ°`-typed code (but not to the extent that enables its evaluation), or otherwise we can simply prove that anything of type `‚ñ°A ‚Üí Bool` is a constant function from the fact that we can‚Äôt do anything on `‚ñ°`-typed stuff (which Tsung-Ju has done), without going through the argument for Rice‚Äôs theorem.
But what operations can we add?
For example, could we have the inverse of Rice‚Äôs theorem, which allows us to add non-trivial but non-extensional operations to the calculus without destroying strong normalisation?

We should of course ask whether a calculus with `GL` is interesting practically, that is, has some uses other than constructing paradoxes.
I mentioned that the fixed-point equation of `GL` has a coinductive feel, and indeed, the resemblance between `GL` and guarded corecursion has been noted by quite some people.
So another aspect worth looking into is the relationship between the dual-modality calculus and guarded type theory.
But still, are there more practical uses of `GL`?

Finally, one reason to have a strongly normalising calculus is to use it as a proof system, which in this case should allow us to internalise Rice‚Äôs theorem, but how this can be done is still rather unclear.
In particular, an equality assumption such as `‚ü¶ cd ‚üß ca = true` should probably be expressed using an internalised term equality (or just reduction) relation on `‚ñ°A`, but this relation is closely related to evaluation and should interact with the latter in some way, about which I sense the danger of self-reference...
